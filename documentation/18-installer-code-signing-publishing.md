# 18 - Installer, Code Signing, And Publishing

## Purpose

This document defines the commercial Windows distribution gate after the
PyInstaller release ZIP is reproducible and validated.

## Installer Manifest

The installer manifest is generated by `release.publishing.create_installer_manifest`
and written to:

```text
release-output/installer-manifest.json
```

It records:

- release ZIP path and SHA-256.
- sidecar release manifest path and SHA-256.
- Windows install root.
- executable name.
- uninstall command.
- Start Menu and Desktop shortcut targets.
- clean Windows smoke commands.
- publishing checklist.
- required signature verification script.

The manifest is metadata only. It must not contain certificates, private keys,
provider API keys, recovery keys, client documents, or cloud credentials.

## Installer Wrapper

F30 standardizes the wrapper boundary before selecting a specific commercial
installer technology.

Required installer behavior:

- install into `%LOCALAPPDATA%\Programs\DocumentVaultIngestionEngine`.
- create a Start Menu shortcut.
- create a Desktop shortcut.
- include an uninstall entry or `uninstall.exe` path.
- preserve portable ZIP distribution as a fallback.
- never write user vault data into the application install folder.

## Code Signing

Signing is mandatory for publishable artifacts, but private signing keys are never
stored in this repository.

Verification script:

```powershell
tools\verify_windows_signature.ps1 -Path <artifact>
```

The script uses `Get-AuthenticodeSignature` only. It does not sign files and does
not require access to a certificate.

## Publishing Checklist

Before publishing:

1. Confirm CI is green on `main`.
2. Build the frozen PyInstaller bundle.
3. Create the release ZIP and sidecar manifest.
4. Generate `installer-manifest.json`.
5. Build the installer from the release ZIP.
6. Sign the installer and release ZIP using the owner-held certificate.
7. Verify signatures with `tools\verify_windows_signature.ps1`.
8. Run portable install validation.
9. Run clean Windows install/uninstall validation.
10. Publish versioned artifacts and release notes with validator evidence.

## Validation

```powershell
python tests\validate_package.py
python tests\validate_frozen_build.py
python tests\validate_release_bundle.py
python tests\validate_portable_install.py
python tests\validate_installer_publishing.py
python tests\validate_security_scan.py
```

## Blockers

Do not publish if:

- signatures are missing or invalid.
- artifact hashes do not match manifests.
- installer writes user vault data into the app install folder.
- release artifacts contain private keys, provider keys, cloud credentials,
  recovery keys, `.env` files, or sample client documents.
- clean Windows `--selftest`, `--products`, `--providers`, and
  `--native-workflow-e2e` do not pass.
